# Milestone 1 ‚Äî Architecture Skeleton & Vertical-Slice Foundations

**Tag:** `v0.1-milestone-1`
**Date:** 2026-05-21
**Scope:** 2-day burst, repository init through vertical-slice content authoring.

## What it is

Corgi Commando is a 2.5D beat-em-up prototype where corgis defend suburbia from feral cats, raccoon bandits, sprinkler turrets, and the WHISKERBOT-9000 boss-mech. This milestone covers the architectural skeleton, every gameplay system stub filled in to spec, and the first vertical-slice content (Sarge the corgi + Level 1 enemies + BackyardWave1).

## What shipped

### Foundations
- **Input abstraction** (#14) ‚Äî timestamped input buffer + Unity PlayerInput bridge; gameplay reads buffered actions, not raw events
- **Entity & components** (#15) ‚Äî composition-based actor base with health, hurtbox, knockback, faction
- **Platform build config** (#20) ‚Äî iOS/macOS validation, IL2CPP/ARM64 requirements, safe-area handling

### Gameplay systems
- **Movement** (#16) ‚Äî kinematic 2.5D controller (X horizontal, Z depth, Y jump/gravity), no Rigidbody2D
- **Combat** (#17) ‚Äî centralized hit resolution: Z-band gating (¬±0.5 units), hitstop, combo tracking with 2s timeout, special meter, AOE multi-target
- **Player Controller** (#22) ‚Äî `CorgiController` FSM with combo chain (Punch ‚Üí Punch ‚Üí Kick) and special-meter integration
- **Enemy AI** (#23) ‚Äî lightweight FSM with aggro slot management (max 2 attackers per target), per-enemy variants for feral cats, raccoon bandits, sprinkler turrets
- **Environmental weapons** (#26) ‚Äî pickup, swing, throw with use counts and movement penalty
- **Boss controller** (#27) ‚Äî WHISKERBOT-9000 three-phase fight (75% / 35% HP thresholds) + Maine Coon pilot eject

### Encounter & co-op
- **Spawn & wave management** (#19) ‚Äî `WaveData`-driven encounter progression
- **Camera** (#18) ‚Äî `GroupTargetCamera` (multi-target framing + distance cap), `ArenaCameraLock` for wave/boss arenas
- **Co-op run state** (#28) ‚Äî shared lives pool, shared Treats, partner revive on proximity

### Content
- **Vertical-slice authoring** (#29) ‚Äî Sarge's `CorgiData`, enemy data for the Level 1 roster, `BackyardWave1` asset, player prefab wired in `Level_Backyard.unity`

### Infrastructure
- **GameCI workflow** (#21) ‚Äî GitHub Actions running EditMode + PlayMode tests on every PR
- **Input handler fix** (#25) ‚Äî `activeInputHandler` set to new Input System; unblocks CI initialization
- **Test isolation** (#36) ‚Äî scene-entity cleanup between EnemyAI tests
- **EditMode orientation injection** (#38) ‚Äî testable platform orientation provider

## Tech story

- **Architecture:** Composition over inheritance for actors (`Entity` + typed components). Lightweight FSMs for AI/player state. ScriptableObject-driven data for tuning without code changes. Custom 2.5D kinematic movement (no Rigidbody2D physics).
- **TDD-first:** Every issue spec ships with explicit `Tests-to-Pass` acceptance criteria. EditMode tests cover ~all gameplay system contracts; PlayMode tests reserved for collision/render-dependent behavior.
- **Copilot-driven authorship:** 19 of the 21 merged PRs were authored by GitHub Copilot agents running in parallel branches. Human/Claude time was concentrated on issue specification, code review, infrastructure (CI, input handler fix), and merge-strategy decisions. See `learnings/copilot-coding-agent-effectiveness.md` for the after-action.
- **Tooling:** Unity 2022.3.62f1 LTS, IL2CPP scripting backend for iOS (ARM64), GameCI v4 for test runs, GitHub Actions for CI.
- **Velocity:** Architecture + 14 gameplay/content PRs landed in two days end-to-end. Critical path was reviewer feedback turnaround, not implementation throughput.
